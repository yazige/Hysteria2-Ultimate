# 任务背景与目标
我有一台基础配置的 VPS（1核 CPU，1G 内存，系统为 Ubuntu 24.04 LTS，网络带宽充裕）。
我需要在这台机器上，为我的团队部署一个极度稳定、支持多用户隔离管理、且免维护的 Hysteria 2 (QUIC 协议) 代理面板。由于机器性能受限，你提供的方案必须是针对该硬件瓶颈的“终极护城河版”。

# 核心架构与硬性优化指标（必须严格集成到脚本的底层逻辑中）
为了防止多人高并发时导致机器死机、断流或内核崩溃，脚本必须无差别集成以下 5 大底层优化：
1. 【内存护城河】强制配置 2GB Swap 虚拟内存，防止流量洪峰导致 OOM (Out Of Memory) 杀掉核心进程。
2. 【CPU 防过载锁】服务端配置文件中，强制将全局上下行总带宽硬性锁定为 300Mbps。绝对不允许跑满物理带宽，以保护单核 CPU 算力，防死机重启。
3. 【UDP 终极防漏包】修改系统 sysctl，将 QUIC 极度依赖的 UDP 读写最大缓冲区（rmem_max/wmem_max）从默认的几百K暴力扩容至 16MB (16777216)。
4. 【无损原生加速】直接调用 Ubuntu 24.04 原生内核强制开启 BBR + FQ 拥塞控制算法。严禁使用第三方内核。
5. 【系统级并发破除】修改 /etc/security/limits.conf 以及 Systemd 全局配置，将文件描述符限制 (Ulimit) 从默认的 1024 强行提升至 1048576，解决 "Too many open files"。

# 综合管理面板与菜单需求（脚本核心交互逻辑）
输出的 Bash 脚本必须是一个交互式的综合管理面板，包含以下 4 个二级菜单选项：
- 【A. 一键无脑部署 (单用户版)】：一键完成底层优化并部署单用户节点，输出小火箭链接及原版完整的 Clash Verge YAML 模板。
- 【B. 多用户独立账号部署】：
  - 允许交互式输入生成的账号数量（默认10，建议≤20）。
  - 自动为每个人生成独立的用户名和随机密码。
  - **核心痛点解决**：输出配置时，必须使用循环，为**每一个用户单独打印一份完整的、包含了分流规则的 Clash YAML 配置文件**（不要将所有人合并在一个代理组里，必须彻底隔离展示，方便我单独发给对应离职/新入职的人员）。
- 【C. 修改/更新多用户密码】：列出当前配置中的所有成员账号并编号，输入编号可重置其密码，并平滑重启服务、重新输出最新链接。
- 【D. 老节点底层网络优化】：针对原本已安装 Hysteria2 但未做底层优化的 VPS，提供无损热升级（补充 Swap、BBR、UDP 16MB 和并发解锁），不破坏原有配置。

# 交互体验与容错要求
1. 在用户选择 A 或 B 执行安装前，必须打印醒目的中文提示：“根据 VPS 的网络情况，全程需要 1 分半到 2 分钟，请耐心等待...”
2. 脚本必须附带【自检清单】，安装/优化完成后，自动验证 Swap、BBR、UDP、Ulimit 等是否真正生效并打印结果（绿勾/红叉）。
3. 使用动态端口跳跃（防火墙 40000 - 50000 端口转发防封锁）。
4. 客户端 Clash YAML 模板内，必须强制将单人客户端限速定死为 `up: 30`, `down: 30`。
